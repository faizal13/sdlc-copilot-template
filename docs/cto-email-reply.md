Yes — and we've gone further than spec-driven development.

Rather than using AI as a writing or autocomplete tool, we've embedded specialised AI agents directly into our delivery pipeline. Each agent has a focused role:

- **Story Refiner** reads a business epic and automatically generates technical child stories, a dependency graph, and a phased execution plan — in minutes, not a half-day meeting
- **Story Analyzer** validates every story before development starts, eliminating mid-sprint blockers from underspecified requirements
- **Dev Agent** implements code following our exact architecture standards and domain rules — not generic code, but code that matches our patterns
- **Reviewer** checks every PR against our architecture rules and acceptance criteria before any human opens it
- **Telemetry Collector** logs every AI session and produces sprint-level reports on usage and quality

The system is wired directly into Azure DevOps — agents read from and write to ADO, so everything stays in one place. Our domain knowledge, architecture decisions, and coding standards are codified and shared across the entire team.
