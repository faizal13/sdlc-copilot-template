# Epic Execution Plans

This directory contains execution plans generated by `@story-refiner`.

Each file follows the naming convention: `EPIC-{id}-execution-plan.md`

## What's in an execution plan?

- **Story-to-service mapping** — which story goes to which microservice
- **Dependency graph** — which stories depend on which
- **Execution phases** — what to build first, what can run in parallel
- **Contract handoffs** — API/event contracts needed between phases
- **Gap report** — missing stories, ambiguous scope, contradictions

## How other agents use these plans

- **@task-planner** reads the execution plan before creating a task plan — it checks if dependencies are met and loads contract handoffs
- **@story-analyzer** reads the execution plan before creating a GitHub Issue — it adds execution context (phase, dependencies, parallel stories)

## When to generate

Run `@story-refiner EPIC-{id}` once per epic, during sprint refinement, before any coding begins.
