The Agents

What Each Agent Does

Five purpose-built agents fire automatically at trigger points. One human gate preserves engineering judgment before code ships.

01
Story Analyzer
Trigger: manual (you run it)

Reads the ADO story and acceptance criteria via MCP, pulls solution-design docs from GitHub, then uses Claude Opus 4.6 to classify the service, identify personas, define state transitions, API contracts, and extract test cases. Creates a structured GitHub Issue.

ADO MCP GitHub MCP Claude Opus 4.6 ~3–5 min
02
Coding Agent
Trigger: ai-generated label on Issue

Checks out the release branch, creates a feature branch, reads the structured GitHub Issue and coding bootstrap guide, then generates the full implementation: Flyway migrations, JPA entities, repositories, service + state machine, controller, OpenAPI spec, and test suites. Raises a Draft PR.

Copilot Workspace Full stack generation ~10–15 min
03
AI Review Agent
Trigger: PR opened

Reads the PR diff against a curated review instructions file. Posts inline comments flagging BigDecimal violations, missing AC test methods, persona isolation gaps, TBD integration placeholders, and missing timeouts — before a human ever opens the PR.

GitHub Actions Inline comments ~3–5 min
🧑
Human Gate
Your engineering judgment

You review the generated code for business logic correctness, address AI review comments, run mvn clean verify locally, then approve and merge. This is the only place human judgment is required — everything else is automated.

Business logic check ~20–40 min → 15 min by Sprint 4+
05
Learning Agent
Trigger: PR merged into release/*

Gets the diff of the merged, human-approved PR. Uses Claude via GitHub Models API (no extra billing on Copilot Enterprise) to extract coding patterns, score confidence, and either promote patterns to .copilot/skills/ (≥0.85 confidence, seen 3×) or stage them in .copilot/instincts/ to build over time.

GitHub Models API Self-learning ~2–3 min
06
ADO Sync Agent
Trigger: push to main

Extracts the ADO story ID from the commit message and calls the ADO REST API to move the story to Done, linking the production commit as evidence. The PO sees the story close automatically with full traceability — zero manual admin.

ADO REST API Auto-close ~1 min

Safety & Architecture

Three Loop Guards (Agent 5)

Agent 5 commits directly to the release branch — no PR raised. Three independent guards make an infinite loop architecturally impossible.

1. Event type mismatch

The workflow triggers on pull_request closed, not push. Direct commits by Agent 5 fire a push event only. The trigger will never match — loop is impossible by design.

2. paths-ignore filter

.copilot/** changes are explicitly ignored in the workflow's paths-ignore rule — even in the unlikely event a PR was raised against those files.

3. Commit message tag

Every learning commit includes [skip-learning] as a final belt-and-braces guard, ensuring the learning workflow will never process its own commits.
