# taskPlan/

Local task plan files — generated by `@task-planner` and consumed by `@local-rakbank-dev-agent`.

## What is this?

This folder is the **local equivalent of GitHub Issues** for the local VS Code workflow.
Instead of creating a remote GitHub Issue (as `@story-analyzer` does), `@task-planner`
writes a structured specification file here that drives local code scaffolding.

## File Naming

| Source | Naming convention | Example |
|--------|-------------------|---------|
| ADO Story | `{ADO-STORY-ID}-{service-name}.md` | `ADO-456-application-service.md` |
| Plain description | `{YYYY-MM-DD}-{kebab-case-summary}.md` | `2026-02-27-mortgage-status-endpoint.md` |

## Lifecycle

```
@task-planner ADO-456
       ↓
taskPlan/ADO-456-application-service.md  ← file created here (Status: ready-for-coding)
       ↓
@local-rakbank-dev-agent taskPlan/ADO-456-application-service.md
       ↓
Code scaffolded in VS Code
       ↓
@local-reviewer  →  @local-instinct-learner  →  git commit
       ↓
docs/ai-usage/sprint-N/ADO-456.md  ← auto-created by post-commit hook
```

## Status Values

Update the `Status` field in the task plan metadata as work progresses:

| Status | Meaning |
|--------|---------|
| `ready-for-coding` | Plan complete — coding agent can start |
| `in-progress` | Coding agent is working / developer is working |
| `in-review` | Code generated — local reviewer running |
| `done` | Committed — ai-usage logged |

## Gitignore Note

Task plan files ARE committed — they are your audit trail and the spec that drove the code.
The git history of `taskPlan/` is traceability evidence for AI-assisted development.
