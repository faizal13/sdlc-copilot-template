# Project Changelog

> **This file is maintained automatically by SDLC agents.** Each entry records a project
> evolution event — requirement changes, design decisions, scope adjustments, and delivery
> milestones. It is the single source of truth for understanding HOW the project evolved
> from initial epic to delivery.
>
> **Rules:**
> - Entries are **append-only** — never edit or delete a previous entry.
> - Each entry is timestamped and tagged with the agent that created it.
> - Re-runs of the same agent on the same epic produce a **Delta** section showing what changed.
> - This file is read by all agents at startup to understand project history.
>
> **Contributing agents:**
> | Agent | When it writes | What it logs |
> |-------|---------------|--------------|
> | `@story-refiner` | Every run | Scope capture, new tasks, requirement drift deltas |
> | `@api-architect` | Every run | Specs generated, revisions, breaking changes |
> | `@task-planner` | Every run | Task breakdown, dependencies, re-planning deltas |
> | `@story-analyzer` | Every run | GitHub Issues created, gaps found |
> | `@local-reviewer` | Only when drift detected | Spec drift, requirement misalignment |
> | `@sprint-orchestrator` | Every run | Sprint snapshot, progress, blockers |
